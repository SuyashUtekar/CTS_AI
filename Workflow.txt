Revised End-to-End Workflow
Stage 1 — Cheque Ingestion
	•	High-speed cheque scanner (Panini/Canon) captures front + back
	•	Stored as TIFF 300 DPI to AWS S3
	•	Event pushed to Kafka topic cheque.ingested

Stage 2 — Image Preprocessing
	•	OpenCV — deskew, denoise, binarization, contrast enhancement
	•	Crop into zones: amount box, date, payee, signature, MICR line
	•	Output clean image patches per field

Stage 3 — OCR Extraction

Output → structured JSON with all extracted fields

Stage 4 — IDP Validation
	•	LayoutLMv3 validates field positions and consistency
	•	Rules engine checks: amount words vs numeric match, stale/post-dated cheque, valid IFSC from RBI master DB
	•	Flags any field-level anomalies

Stage 5 — Risk Scoring


Output → Risk score 0–100 with SHAP explainability

Stage 6 — Agentic Decision Engine
	•	LangGraph orchestrates the full pipeline as an agent
	•	Claude Sonnet reasons over risk score + validation flags
	•	Decision logic:

0–30   → Auto Approve
31–70  → Secondary checks
71–100 → Manual review queue

Stage 7 — Manual Review (High Risk Only)
	•	Temporal.io manages human-in-the-loop queue
	•	React dashboard shows cheque image + AI reasoning summary
	•	2-hour SLA with auto-escalation

Stage 8 — NGCB Export
	•	Structured data + risk score pushed to banking system
	•	Apache Camel handles ISO 8583 / CTS XML formatting
	•	mTLS secured, HSM-signed

Stage 9 — NPCI Submission
	•	Verified cheque bundle (TIFF + XML) sent to NPCI via CTS protocol
	•	Python reconciliation agent handles ACK/NACK
	•	Cleared funds reflect in accounts

One-Line Flow

Scan → Preprocess → PaddleOCR-VL-1.5 + Textract(MICR) 
→ IDP Validate → Risk Score → LangGraph Agent 
→ Auto Approve / Manual Review → NGCB → NPCI

Complete Flow Summary

Physical Cheque
      ↓
[S1] Ingestion      → cheque_id + S3 paths
      ↓
[S2] Preprocessing  → cropped zone images
      ↓
[S3] OCR            → structured JSON fields
      ↓
[S4] IDP Validate   → PASS/FAIL + flags
      ↓
[S5] Risk Score     → score 0-100 + explanations
      ↓
[S6] Agent Decision → AUTO_APPROVE / MANUAL_REVIEW / REJECT
      ↓              ↓
[S7] Human Review  (if flagged)
      ↓
[S8] NGCB Export    → CTS XML
      ↓
[S9] NPCI Submit    → ACK/NACK + clearing date
